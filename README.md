# デジフェス 配信グラフィック（dezifes-nodecg）

デジ窓フェスティバル（デジフェス）のスプラトゥーン 3 大会用 NodeCG 配信グラフィックシステムです。ナワバリトーナメント・エリアトーナメントのチーム情報を管理・表示します。

## 会場 PC セットアップ手順（Windows 11）

開発環境が入っていない素の Windows PC で動かすための手順です。

### 1. このリポジトリの準備

#### Git がインストール済みの場合

```powershell
git clone <このリポジトリのURL>
cd dezifes-node
```

#### Git がない場合

1. リポジトリの ZIP をダウンロードして展開
2. PowerShell で展開先のフォルダに移動

```powershell
cd C:\Users\<ユーザー名>\Downloads\dezifes-node
```

### 2. Node.js のインストール

1. https://nodejs.org/ にアクセスし、**v24 LTS**（推奨版）のインストーラーをダウンロード
2. インストーラーを実行（設定はすべてデフォルトのまま「Next」で OK）
3. インストール完了後、**PowerShell** を開いて以下を実行し、バージョンが表示されれば成功

```powershell
node -v
# v24.x.x と表示されれば OK
```

### 3. pnpm の有効化

Node.js に同梱されている Corepack で pnpm を有効化します。PowerShell で実行してください。

```powershell
corepack enable
```

> **エラーが出る場合**: PowerShell を **管理者として実行** してからもう一度 `corepack enable` を試してください。

> それでも実行できない場合こちらを参照。
> https://qiita.com/araiWorks/items/6964e85a73bff3ff705c

### 4. 依存関係のインストール

```powershell
pnpm install
```

初回は数分かかります。

### 5. ビルド

```powershell
pnpm build
```

エラーなく完了すれば準備完了です。

### 6. 各種ファイルの配置

`data` フォルダ内に、Discord 上のリンクから落としてきたデータをコピーします。

### 7. 起動

```powershell
npx nodecg start
```

起動後、ブラウザで http://localhost:9090 にアクセスします。

### 8. OBS への取り込み

1. 開いたサイトで右上の「GRAPHICS」をクリック
2. 使用するグラフィックの「COPY URL」をクリック
3. OBS で「ソース」→「＋」→「ブラウザ」を追加
4. URL に上記の Graphics URL を入力
5. 幅 `1920`、高さ `1080` に設定
6. 「カスタム CSS」は空にする（デフォルトの body 背景色指定を消すため）

#### グラフィック一覧

| グラフィック名 | 用途 |
|---|---|
| `under` | 配信画面下部のチーム名オーバーレイ（ナワバリ/エリア共通） |
| `side` | 会場スクリーン左右分割表示（ナワバリ/エリア共通） |
| `champions` | デジ窓チャンピオンズ用 |
| `cast` | 実況・解説担当者表示（メインレイアウト） |
| `cast-replay` | 実況・解説担当者表示（リプレイレイアウト） |

## 運営操作ガイド

### ダッシュボードの構成

ダッシュボードには 4 つのタブ（Workspace）があります。

- **BATTLE** — チーム選択・プレビュー編集・表示操作パネル
- **判定結果** — OCR 自動判定結果の確認・手動入力画面
- **設定** — チーム情報 CSV の再読込・アップロード / Google スプレッドシート同期 / 担当者リスト読込
- **caster** — 実況・解説担当者の選択・表示操作

### 試合前 - 基本的な操作の流れ

1. **モード切替** で「ナワバリ」または「エリア」を選択する。**切り替え時に表示状態は自動リセットされるため注意。**
2. **チーム選択** パネルでアルファ・ブラボーのチームを選ぶ
3. **プレビュー編集** パネルで内容を確認（必要なら「編集」ボタンで修正）
4. チーム入場時、 **表示操作** パネルで「アルファチーム表示」「ブラボーチーム表示」を押す → 配信画面にフェードインで表示される
5. もう一度同じボタンを押すとフェードアウトで非表示になる
6. 試合終了後、 **リセットボタン** を押すと、両チーム非表示 + 選択も初期化される

### 試合中・試合後 - 判定結果パネルの使い方

試合開始後、OBS から自動的にスクリーンショットが送信され、ブキ編成とステージが自動入力されます。

1. **判定結果タブ** でプレイヤー名・ブキを確認し、誤りがあれば修正
2. ステージ名をドロップダウンから選択（自動判定がある場合は自動入力される）
3. 勝利チームの「勝利」ボタンを押す（自動判定がある場合は自動入力される）
4. 「確定して記録」を押すと `records.csv`（および Google スプレッドシート）に記録される

> **画像 が送信されなかった場合**: リプレイ保存フォルダに格納されている PNG ファイルをドロップゾーンにドラッグ＆ドロップするか、「**手動で入力**」ボタンを押して手動入力できます。

### チーム情報 CSV の更新

設定タブの「CSV 再読込・アップロード」パネルには 2 つの方法があります。

| 操作 | 説明 |
|---|---|
| **チーム情報 CSV 再読込** ボタン | サーバー上の `data/teams.csv` を再読み込み。**編集内容はすべて破棄**されます |
| **参照…** ボタン（ファイル選択） | ローカル PC から CSV ファイルを選択してアップロード。再読込と同じく、編集内容は上書きされます |

## OBS 連携エンドポイント

OBS から NodeCG に HTTP リクエストを送ることで、試合情報（ブキ編成・ステージ・勝敗）を自動入力できます。

**ベース URL**: `http://localhost:9090`

| エンドポイント | メソッド | Content-Type | 用途 |
|---|---|---|---|
| `/weapons` | POST | `text/plain` | ブキ編成スクリーンショットを送信し OCR を実行 |
| `/stage` | POST | `text/plain` | ステージ画面スクリーンショットを送信しステージを自動判別 |
| `/result` | POST | `application/json` | 試合結果（勝利サイド）を送信 |

### POST /weapons

試合後のリザルト画面（ブキ編成が表示されている画面）のスクリーンショットを送信します。Extension が OCR を実行し、プレイヤー名・ブキ編成を「判定結果」タブに自動入力します。

- **Body**: スクリーンショット PNG の Base64 エンコード文字列（上限 10 MB）
- **レスポンス**: `202 Accepted`（OCR 処理は非同期で実行されます）

### POST /stage

試合開始直後のステージ確認画面のスクリーンショットを送信します。`data/stages/` に格納されたテンプレート画像と比較（ZNCC 法）し、ステージ名を自動判別して「判定結果」タブのドロップダウンに反映します。

- **Body**: スクリーンショット PNG の Base64 エンコード文字列（上限 10 MB）
- **レスポンス**: `202 Accepted`（マッチング処理は非同期で実行されます）

> `/weapons` より先に送信するのが理想ですが、後着でも直近の OCR 候補にステージ名を後付けで反映します。

### POST /result

試合終了後に勝利サイドを送信します。「判定結果」タブの「勝利」欄に自動入力されます（Dashboard での手動変更も可能です）。

- **Body**（JSON）:
  ```json
  { "result": "alpha_win" }
  ```
  または
  ```json
  { "result": "bravo_win" }
  ```
- **レスポンス**: `202 Accepted`

`result` の値は `alpha_win`（アルファチーム勝利）または `bravo_win`（ブラボーチーム勝利）のいずれかです。

### OBS での設定例

OBS の「高機能シーンスイッチャー」プラグインで上記 URL に POST リクエストを送信します。

## トラブルシューティング

### `pnpm` コマンドが見つからない

```powershell
corepack enable
```

を管理者権限の PowerShell で実行してください。

### ビルドでエラーが出る

Node.js のバージョンが v22 以上であることを確認してください。

```powershell
node -v
```

### 起動後にブラウザでアクセスできない

- `npx nodecg start` が正常に起動しているか確認
- Windows ファイアウォールのダイアログが出たら「許可」を選択
- URL が `http://localhost:9090` であることを確認（`https` ではない）

### 同じ LAN 内の別 PC からアクセスしたい

起動 PC の IP アドレスを確認し、`http://<IPアドレス>:9090` でアクセスできます。

```powershell
ipconfig
```

で IPv4 アドレスを確認してください（例: `192.168.1.100`）。
