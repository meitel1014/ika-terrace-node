## teams.csv 仕様（フォールバック用）

| 列名 | 内容 |
|---|---|
| `チーム名` | チーム名原本。編集禁止。Dashboard 表示で使用 |
| `チーム名(表示用)` | Graphic 表示用チーム名。`<br>` 等の HTML タグを含められる。空の場合は `チーム名` にフォールバック |
| `プレイヤー1`〜`プレイヤー4` | プレイヤー名。列名に数字（半角 `1`〜`4` / 全角 `１`〜`４` / 丸数字 `①`〜`④`）を含めば認識される |

チーム ID は読み込み順の連番文字列（`"1"`, `"2"`, ...）で自動付与される。チーム名の重複チェックは行わない。
CSV由来のチームは `xId` / `weapons` が空になる（スプレッドシートにしか無い情報のため）。

## Googleスプレッドシート連携仕様

チーム情報は以下2つのスプレッドシートを読み合わせて構築する（CSVより優先）。

- **申請フォームの回答**（大会ごとにURLが変わる）: 列 `チーム名`, `プレイヤーNの名前`, `プレイヤーNのX(Twitter)ID`（N=1〜4）
- **選手情報登録フォームの回答**（固定）: 列 `プレイヤー名`, `X(Twitter) ID`, `持ちブキを選択してください（最大3つまで）`

突合ロジック: 申請フォームの「プレイヤーNの名前」+「プレイヤーNのX ID」をキーに、選手情報登録フォームの行を検索し、一致した行の「持ちブキ」を取得する。

- 名前は前後空白をtrimして比較
- X IDは先頭 `@` の有無・大文字小文字を無視して正規化してから比較
- 持ちブキはカンマ等（`,` `、` `，`）区切りで1セルに複数格納されている前提でパースし、`weapon_aliases.csv` の `ja`列と照合して `id`（画像ファイル名）に変換する。マッチしない武器名はログ警告のうえ無視する

### `data/sheets.json` の形式

```json
{
  "applicationForm": {
    "url": "https://docs.google.com/spreadsheets/d/XXXXXXXX/edit",
    "sheetTitle": "フォームの回答 1"
  },
  "playerRegistrationForm": {
    "url": "https://docs.google.com/spreadsheets/d/XXXXXXXX/edit",
    "sheetTitle": "フォームの回答 1"
  }
}
```

`url` はスプレッドシートのURLをそのまま貼り付ければよい（IDを正規表現で自動抽出する）。`sheetTitle` はワークシート（タブ）名。省略時・不一致時は先頭シートにフォールバックする。

### サービスアカウントの準備（よくある詰まりポイント）

1. GCPでサービスアカウントを作成し、鍵（JSON）をダウンロードして `data/credentials/` 配下に配置する
2. 鍵JSON内の `client_email` を、対象スプレッドシート（申請フォーム・選手情報登録フォーム両方）に**閲覧者として共有**する（共有を忘れると読み込みに失敗し、CSVへフォールバックする）
3. Google Sheets API がGCPプロジェクトで有効化されていることを確認する

読み込みに失敗した場合（鍵未配置・共有忘れ・API未有効化等）は `teams.csv` に自動フォールバックする。Dashboardの「設定｜チーム情報再読込」パネルから手動再読込もできる。
